# Retro — 2026-03-03 Security-Key Milestone

## Scope
- Period: 2026-02-27 to 2026-03-03
- Focus: YubiKit migration, OpenPGP/PIV/FIDO2 reliability, backup/export/import, ProxyJump, password save

## What Went Well
- Debug-report pipeline enabled rapid root-cause analysis without ADB/logcat.
- High-risk auth regressions were isolated by structured flow markers (`PK_OK`, `SIGN_REQUESTED`, `SK_SIGN_*`).
- Team delivered cross-provider support (OpenPGP/PIV/FIDO2) with reproducible Docker builds.
- User validation loop was fast and high quality; every fix got real-device confirmation.

## What Went Wrong
- External plugin edits introduced major regressions without preserving lifecycle assumptions.
- YubiKit connection-lifecycle behavior (`requestConnection` callback scope) was not documented early enough.
- OpenPGP auth had multiple nuanced failures (PIN mode mismatch, stale connection patterns) that consumed cycles.
- UI consistency lagged behind functional progress (PIN input differences, weak "touch/insert key" cue).

## Root Lessons
- NFC/security-key flows need strict connection-lifetime rules and explicit architecture guardrails.
- Auth observability must stay first-class; otherwise regressions are expensive to triage on mobile-only devices.
- UI/UX should be tracked as first-order technical debt during security feature delivery.
